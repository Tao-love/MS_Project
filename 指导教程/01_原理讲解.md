# 小车项目原理讲解

## 1. 项目目标

这个课设项目的目标很直接：

- 用 `STM32L432KC` 作为主控
- 用超声波模块测前方距离
- 用 `L298N` 驱动左右两个直流电机
- 用 OLED 显示实时距离
- 用有源蜂鸣器做障碍报警

当前这版代码实现的是一个最基础的避障流程：

- `distance > 20cm`：小车前进，蜂鸣器不响
- `10cm < distance <= 20cm`：蜂鸣器响，小车缓慢停车
- `distance <= 10cm`：蜂鸣器继续响，小车快速停车

这不是完整的智能避障车，只是先把“测距 + 行动 + 报警”这条主链路打通。

## 2. 系统整体思路

整个系统可以拆成 4 个部分：

1. 测距部分
2. 电机驱动部分
3. 报警部分
4. 显示部分

它们之间的关系是：

- 超声波先测出距离
- 主循环读取距离值并判断区间
- 根据区间调用电机函数和蜂鸣器函数
- OLED 把当前距离显示出来

## 3. 超声波测距原理

本项目使用的是常见超声波模块，核心过程是：

1. `Trig` 发一个触发脉冲
2. 超声波模块开始发射声波
3. 遇到障碍物后回波返回
4. `Echo` 输出一个高电平脉冲
5. 高电平持续时间和距离成正比

代码中用 `TIM1` 输入捕获来测这个高电平脉冲宽度：

- 上升沿时记下开始计数值
- 下降沿时记下结束计数值
- 两者相减得到时间差
- 再换算成距离

公式是：

```text
distance = time_us * 0.034 / 2
```

原因是：

- 声速大约 `0.034 cm/us`
- 超声波走的是“去 + 回”两段路
- 所以最后要除以 2

## 4. 为什么超声波中断不会让电机 PWM 停掉

这是这个项目里很重要的一个理解点。

电机 PWM 用的是 `TIM2`，超声波输入捕获用的是 `TIM1`，两者不是同一个定时器。

所以：

- `TIM2` 会按硬件自动输出 PWM
- `TIM1` 只是在捕获回波时进一次中断
- 中断里只是读计数值、算距离

只要你没有在中断里主动去改 `TIM2` 的比较值，电机 PWM 不会因为超声波中断自动停掉。

也就是说：

- `TIM1` 负责“测”
- `TIM2` 负责“转”

它们是并行工作的。

## 5. 电机驱动原理

本项目用的是 `L298N` 电机驱动板，代码控制方式是最常见的：

- `ENA / ENB`：接 PWM，用来控速
- `IN1 / IN2`：控制左轮方向和刹车
- `IN3 / IN4`：控制右轮方向和刹车

### 5.1 方向控制

一个电机常见状态如下：

- 正转：一个输入高，一个输入低
- 反转：一个输入低，一个输入高
- 缓停：两个输入都低
- 快停：两个输入都高

在你现在这版代码里：

- 左轮用 `IN1` 和 `IN2`
- 右轮用 `IN3` 和 `IN4`

### 5.2 调速控制

速度不是靠 `IN1~IN4` 控的，而是靠 `ENA/ENB` 上的 PWM 控的。

PWM 的意思可以简单理解成：

- 高频率开开关关
- 一个周期内“导通时间占比”越大，平均电压越高
- 电机平均得到的能量越多，转得越快

当前工程里：

- `TIM2_CH1 -> ENA`
- `TIM2_CH2 -> ENB`
- PWM 频率是 `1kHz`

初始化速度宏目前写的是：

- 左轮 `speed_L_init = 50`
- 右轮 `speed_R_init = 50`

由于当前 `TIM2` 的周期是 `100 - 1`，所以比较值 50 大致就是 50% 占空比。

## 6. 蜂鸣器原理

你现在接的是有源蜂鸣器，而且实测是“低电平触发”。

这意味着：

- GPIO 输出低电平：响
- GPIO 输出高电平：不响

所以代码里：

- `Buzzer_On()` 写的是 `GPIO_PIN_RESET`
- `Buzzer_Off()` 写的是 `GPIO_PIN_SET`

如果以后发现逻辑反了，先不要急着怀疑代码，先确认模块是不是低电平触发，以及 `IN`、`VCC`、`GND` 有没有接对。

## 7. OLED 显示原理

OLED 在这个项目里只做一件事：显示距离。

流程很简单：

1. 初始化 OLED
2. 每次主循环刷新一帧
3. 把距离格式化成字符串
4. 显示到屏幕上

它不参与控制决策，只负责把当前状态展示出来。

## 8. 当前避障策略的意义

现在这版策略不是“智能路径规划”，而是非常基础的阈值控制：

- 大于 `20cm`：说明前面相对安全，继续前进
- `10cm ~ 20cm`：说明接近障碍物，先报警并停下来
- 小于等于 `10cm`：说明已经非常近，用更强制的方式停车

这个策略的优点是：

- 好理解
- 好调试
- 适合课设

缺点也很明显：

- 不会绕障
- 不会倒车
- 不会转向
- 阈值一旦不合适，就容易“停太早”或“停太晚”

所以它更适合作为第一阶段实现。

## 9. 为什么接线乱也能做

从课设角度看，实物接线不一定漂亮，但只要逻辑关系对，系统就能工作。

真正要重点看的不是线排得多整齐，而是下面这三件事有没有做对：

1. `STM32` 的哪个脚接到了哪个模块输入
2. 所有模块有没有共地
3. 电源和信号有没有接反

你这次文档里会单独给出“STM32 对应接线表”，那个表比实物图更重要。

## 10. 后续如果要继续扩展

这个项目后面最自然的扩展方向有：

- 把“接近障碍物就停”改成“接近障碍物就减速”
- 加倒车逻辑
- 加左右转向逻辑
- 把蜂鸣器改成间歇报警
- 在 OLED 上增加状态文字，比如 `RUN / STOP / ALERT`

但就当前课设阶段来说，现有结构已经足够继续往下改。
