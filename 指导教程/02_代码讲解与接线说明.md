# 小车项目代码讲解与接线说明

- `IMG_20260526_003031.jpg`

## 2. 当前工程里的核心文件

这个项目目前主要看下面几个文件：

- `Core/Inc/main.h`
- `Core/Src/main.c`
- `Core/Inc/motor.h`
- `Core/Src/motor.c`
- `Core/Inc/buzzer.h`
- `Core/Src/buzzer.c`
- `Core/Src/gpio.c`
- `Core/Src/tim.c`

模块职责如下：

- `main.c`：主流程，测距判断，调用电机和蜂鸣器
- `motor.c/.h`：左右轮前进、后退、停车、调速
- `buzzer.c/.h`：蜂鸣器开和关
- `gpio.c`：普通 GPIO 输出口初始化
- `tim.c`：超声波输入捕获和电机 PWM 初始化

## 3. STM32 对应接线表

下面这张表是最重要的。

### 3.1 电机驱动 L298N

| STM32 引脚 | 代码名 | 接到 L298N | 作用 |
| --- | --- | --- | --- |
| `PA0` | `ENA` | `ENA` | 左轮 PWM 调速 |
| `PA1` | `ENB` | `ENB` | 右轮 PWM 调速 |
| `PB0` | `IN1` | `IN1` | 左轮方向控制 1 |
| `PB1` | `IN2` | `IN2` | 左轮方向控制 2 |
| `PB3` | `IN3` | `IN3` | 右轮方向控制 1 |
| `PB4` | `IN4` | `IN4` | 右轮方向控制 2 |

### 3.2 超声波模块

| STM32 引脚 | 代码名 | 接到超声波模块 | 作用 |
| --- | --- | --- | --- |
| `PA11` | `Trig` | `Trig` | 发送触发脉冲 |
| `PA8` | `Echo` | `Echo` | 接收回波脉冲 |

### 3.3 蜂鸣器

| STM32 引脚 | 代码名 | 接到蜂鸣器模块 | 作用 |
| --- | --- | --- | --- |
| `PA6` | `alarm` | `IN` | 报警控制 |

说明：

- 这个蜂鸣器是低电平触发
- 所以 `PA6` 输出低电平时会响

### 3.4 OLED

OLED 走的是 `I2C1`，当前工程保留了 OLED 显示。

常见连接方式是：

- `PA9` -> `SCL`
- `PA10` -> `SDA`

如果你的 OLED 模块丝印不同，以模块丝印为准。

## 4. 电源和共地要求

这个部分比“信号线接哪根”还重要。

必须满足：

1. `STM32 GND` 要接 `L298N GND`
2. 电池负极也要接到公共 `GND`
3. 超声波模块和蜂鸣器也要共地

一句话总结：

- 所有模块最终都必须共地

如果没有共地，最常见的问题就是：

- PWM 看起来有输出，但电机不正常
- 超声波数值乱跳
- 蜂鸣器逻辑反常

## 5. L298N 电机输出怎么接

L298N 的输出端一般是两组：

- `OUT1 / OUT2`
- `OUT3 / OUT4`

接法是：

- 左边电机两根线接一组输出
- 右边电机两根线接另一组输出

例如：

- 左轮电机 -> `OUT1 / OUT2`
- 右轮电机 -> `OUT3 / OUT4`

如果上电后发现“前进命令下轮子反转”，不要先改代码，先直接把对应电机的两根线对调，处理最快。

## 7. `motor.h` 和 `motor.c` 怎么看

这一版 `motor` 模块写得很直接，适合后续自己改。

### 7.1 宏定义

```c
#define MOTOR1 1
#define MOTOR2 2
#define speed_L_init 50
#define speed_R_init 50
```

含义是：

- `MOTOR1` 表示左轮
- `MOTOR2` 表示右轮
- 左右轮初始化占空比先给 `50`

### 7.2 `Motor_Init()`

这个函数做了 3 件事：

1. 启动 `TIM2` 的两个 PWM 通道
2. 设置左右轮初始速度
3. 让左右轮先进入慢停状态

所以程序刚启动时不会直接往前冲。

### 7.3 `Motor_SetSpeed()`

这个函数的作用是给指定电机设置 PWM 比较值。

当前逻辑是：

- 如果传入 `MOTOR1`，就改 `TIM2_CH1`
- 如果传入 `MOTOR2`，就改 `TIM2_CH2`
- 如果速度超过当前 PWM 周期最大值，就截断到最大值

也就是说：

- 左轮速度由 `ENA`
- 右轮速度由 `ENB`

控制。

### 7.4 `Motor_Forward()`

逻辑很简单：

- `MOTOR1`：`IN1=1, IN2=0`
- `MOTOR2`：`IN3=1, IN4=0`

这表示两个轮子都按“前进方向”输出。

### 7.5 `Motor_Reverse()`

逻辑是：

- `MOTOR1`：`IN1=0, IN2=1`
- `MOTOR2`：`IN3=0, IN4=1`

这时轮子反转。

### 7.6 `Motor_SlowStop()`

逻辑是：

- `MOTOR1`：`IN1=0, IN2=0`
- `MOTOR2`：`IN3=0, IN4=0`

这可以理解成断开驱动，让轮子自然停下。

### 7.7 `Motor_FastStop()`

逻辑是：

- `MOTOR1`：`IN1=1, IN2=1`
- `MOTOR2`：`IN3=1, IN4=1`

这属于更强制的刹车方式。

### 7.8 `Car_Forward()` 等整车函数

这些函数只是对左右轮函数做了一层打包：

- `Car_Forward()`：左右轮都前进
- `Car_Reverse()`：左右轮都后退
- `Car_SlowStop()`：左右轮都慢停
- `Car_FastStop()`：左右轮都快停

好处是主函数里调用时更直观。

## 8. `buzzer.h` 和 `buzzer.c` 怎么看

这个模块非常基础，只有 3 个函数：

- `Buzzer_Init()`
- `Buzzer_On()`
- `Buzzer_Off()`

关键点只有一个：

- 这是低电平触发蜂鸣器

所以：

- `Buzzer_On()` 实际写低电平
- `Buzzer_Off()` 实际写高电平

如果以后换成高电平触发模块，就把这两个函数的高低电平反过来即可。

## 9. `tim.c` 里面的两个定时器

### 9.1 `TIM1`

`TIM1` 用来做超声波输入捕获：

- 分频：`72 - 1`
- 周期：`65535`

这里的目的不是输出 PWM，而是测 `Echo` 脉冲宽度。

### 9.2 `TIM2`

`TIM2` 用来做左右轮 PWM：

- 分频：`720 - 1`
- 周期：`100 - 1`

在当前时钟配置下，这一组参数对应的是大约 `1kHz` PWM。

这也是你前面确认过的那个频率。

## 10. `main.c` 是怎么把各模块串起来的

### 10.1 初始化阶段

`main()` 里初始化顺序是：

1. `MX_GPIO_Init()`
2. `MX_I2C1_Init()`
3. `MX_TIM1_Init()`
4. `MX_TIM2_Init()`
5. `OLED_Init()`
6. `Motor_Init()`
7. `Buzzer_Init()`
8. 启动超声波相关定时器

这说明：

- GPIO 先准备好
- 定时器先准备好
- 然后再让各模块进入工作状态

### 10.2 测距中断回调

`HAL_TIM_IC_CaptureCallback()` 做的是：

- 读上升沿时间
- 读下降沿时间
- 计算时间差
- 换算成 `distance`

这个 `distance` 是全局变量，主循环里直接用它做判断。

### 10.3 主循环逻辑

主循环目前做的事情顺序是：

1. 给 `Trig` 发触发脉冲
2. 等待测距结果更新
3. 根据距离判断小车动作
4. 刷新 OLED 显示

当前判断条件是：

```c
if (distance > 20.0f)
{
    Buzzer_Off();
    Car_Forward();
}
else if (distance > 10.0f)
{
    Buzzer_On();
    Car_SlowStop();
}
else
{
    Buzzer_On();
    Car_FastStop();
}
```

所以这版代码不是“边靠近边减速前进”，而是：

- 远了就走
- 近了就报警并停
- 更近就强制停

## 11. 想自己调参数时，改哪里

### 11.1 改默认速度

改这里：

```c
#define speed_L_init 50
#define speed_R_init 50
```

位置在：

- `Core/Inc/motor.h`

### 11.2 改运行时速度

调用：

```c
Motor_SetSpeed(MOTOR1, xxx);
Motor_SetSpeed(MOTOR2, xxx);
```

其中 `xxx` 就是 PWM 比较值。

当前 `TIM2` 周期接近 100，所以：

- `25` 大约是 25%
- `50` 大约是 50%
- `75` 大约是 75%

### 11.3 改 PWM 频率

改：

- `Core/Src/tim.c`

看 `MX_TIM2_Init()` 里的：

- `Prescaler`
- `Period`

你前面已经确认过，现在就是 `1kHz`。

### 11.4 改避障阈值

改：

- `Core/Src/main.c`

也就是下面这两个阈值：

- `20.0f`
- `10.0f`

## 12. 调试时最容易出问题的地方

最常见问题如下：

### 12.1 蜂鸣器该响不响，不该响时反而响

优先检查：

- 蜂鸣器模块是不是低电平触发
- `IN` 有没有接到 `PA6`
- `GND` 有没有共地

### 12.2 电机能响但不转，或者转得很怪

优先检查：

- `ENA/ENB` 有没有真的接 PWM 输出脚
- `IN1~IN4` 有没有接错顺序
- 电池电压够不够
- `STM32 GND`、驱动板 `GND`、电池负极是不是共地

### 12.3 方向反了

最简单的处理方法：

- 对调对应电机的两根输出线

### 12.4 超声波数值不稳定

优先检查：

- `Trig` 和 `Echo` 是否接反
- 供电是否稳定
- 地线是否可靠
- 车体运动时是不是线太松
